# Парсер gdebenz.ru

`gdebenz_parser.py` собирает АЗС из публичных JSON-ручек карты gdebenz.ru и сохраняет CSV + JSON.

## Единый файл по Московской области

```bash
python3 outputs/gdebenz_parser.py \
  --all \
  --with-status \
  --with-real-count \
  --with-districts \
  --workers 24 \
  --insecure-ssl
```

Итоговые файлы:

- `gdebenz_unified_status_realcount_districts.csv`
- `gdebenz_unified_status_realcount_districts.json`

Во время выполнения парсер показывает полоски прогресса по этапам: сбор точек, статусы, `realCount`, городские округа.
В файлы добавляется колонка `exported_at` - время формирования выгрузки, его показывает дашборд.

## Колонки

- `region` - Московская область
- `exported_at` - когда сформирована выгрузка
- `district` - городской округ
- `station_name` - название АЗС
- `brand` - бренд
- `lat`, `lon` - координаты
- `osm_id` - идентификатор точки
- `status` - русский текст статуса
- `status_code` - исходный код сайта
- `fuels_now` - исходная строка топлива
- `92`, `95`, `98`, `100`, `ДТ` - отдельные признаки топлива: `1`, если вид топлива указан
- `confirmations` - подтверждения из gdebenz.ru
- `realCount` - количество последних отметок водителей за 24 часа
- `last_at` - время последней отметки

## Перевод статусов

- `no` -> `нет бензина`
- `queue` -> `есть бензин но очередь`
- `yes` -> `есть бензин`
- `low` -> `мало бензина`

## Примечание

Для Московской области исходный сбор идёт по прямоугольнику, а затем строки фильтруются по региону после геокодирования. Поэтому в финальном едином файле остаётся только `Московская область`.

## Как разместить онлайн

Самый простой вариант - GitHub Pages:

1. Создайте репозиторий на GitHub и загрузите туда папку проекта.
2. В настройках репозитория откройте `Pages`.
3. В `Build and deployment` выберите источник `GitHub Actions`.
4. Откройте `Actions -> Update gdebenz data -> Run workflow`.
5. После выполнения откройте опубликованный адрес `gdebenz_dashboard.html`.

Файл `.github/workflows/update-gdebenz.yml` уже настроен на автоматическое обновление каждые 30 минут, коммит новых CSV/JSON и публикацию папки `outputs` на GitHub Pages.
